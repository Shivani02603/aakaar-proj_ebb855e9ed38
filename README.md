## 🚀 Quick Start (60 seconds, zero configuration)

> **Run everything from THIS folder (the project root) — not from backend/ or frontend/.**

**Windows (PowerShell):**
```powershell
.\run_backend.ps1        # terminal 1 — creates venv, installs deps, starts the API
.\run_frontend.ps1       # terminal 2 — installs deps, starts the UI
```

**macOS / Linux:**
```bash
bash run_backend.sh      # terminal 1
bash run_frontend.sh     # terminal 2
```

Open http://localhost:5173 — with no `.env` present, the backend starts in **DEMO mode**
(SQLite, no keys needed) so you can try the app immediately.

## Real configuration

1. `cp .env.example .env` and fill in `DATABASE_URL` (PostgreSQL), `JWT_SECRET`
   (and API keys if this is an AI app — see the comments inside `.env.example`,
   which include ready-made settings for OpenAI, Groq, and Gemini).
2. Run the same scripts again. The `.env` file is loaded automatically.

## Verify it works (hermetic — no DB, no keys)

```bash
./venv/Scripts/python -m pytest tests/ -q     # Windows
./venv/bin/python -m pytest tests/ -q         # macOS/Linux
```

## Manual commands (if you prefer no scripts)

```bash
# from the PROJECT ROOT:
python -m venv venv && venv/Scripts/activate      # or source venv/bin/activate
pip install -r requirements.txt
python -m uvicorn backend.main:app --port 8000    # use `python -m uvicorn`, not bare uvicorn
cd frontend && npm install && npm run dev
```

## Or with Docker

```bash
docker compose up --build
```

## Layout
- `backend/routers/` — API routers, auto-mounted at `/api` (a file exposing `router` is live).
- `database/models.py`, `backend/schemas.py`, `frontend/src/types.ts` — stamped from ONE
  contract; keep them in sync by regenerating, not hand-editing.
- `tests/` — acceptance tests; these were green before this repo was pushed.
